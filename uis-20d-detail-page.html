Work Order Detail Page

Detail page for Work Orders using DataModelDetails with custom renderers for line items and signatures on the Overview tab. Summary cards, Invoices tab, PDF download template, and header actions.

Estimate: 4-6 days Requires: WO-3a/b/c/d, WO-2 Enables: WO-5 (WO Log), WO-7 (Conversion), WO-8 (Mobile)

1. Tab Structure

Line items and signatures are part of Overview, not separate tabs.

TabContentNotes
OverviewSummary cards + DataModelDetails with customColumnRender for labor/material/equipment/signaturesDefault tab. All WO data in one view.
InvoicesReuses JobBillList from Job detail Billing tabNo create button — creation via header "Generate Invoice" (WO-7). Read-only list.
FilesExisting Job file tabNo changes.
SettingsDataModelSettings with subModel="WorkOrder"WO-specific settings.

3. Overview Tab

Summary Cards (2×3 grid)

CardSourcePermission
Total ValueLabor + material + equipment subtotalsviewWageDetails (shows "—" otherwise)
Labor HoursSum of TimeSheetDetailRecord hours (structured) + WorkOrderLaborItem durations × qty (blank lines)Always visible
Material CostSum of material totalsAlways visible
EquipmentSum of equipment durationsAlways visible
StatusCurrent WO status badgeAlways visible
Date of WorkJob.dateOfWorkAlways visible

DataModelDetails

<DataModelDetails
    modelName="Job"
    recordId={jobId}
    customColumnRender={{
        "wo-labor-items": () => <LaborItemsDisplay jobId={jobId} />,
        "wo-material-items": () => <MaterialItemsDisplay jobId={jobId} />,
        "wo-equipment-items": () => <EquipmentItemsDisplay jobId={jobId} />,
        "wo-signatures": () => <SignaturesDisplay jobId={jobId} />
    }}
    viewSectionType={"DETAILS" as GqlViewSectionType}
/>

Custom display components (read-only)

  • LaborItemsDisplay: Combined read-only table from two sources. Timesheet entries: employee name, trade level, rate label hours breakdown (from TimeSheetDetailRecords for the job). Blank rows: description + hours + rate (from WorkOrderLaborItems). Rates only with viewWageDetails. Subtotal combines both sources.
  • MaterialItemsDisplay: Table: material, qty, unit cost, total + subtotal.
  • EquipmentItemsDisplay: Combined read-only table from two sources. Logged equipment: equipment name, rate type, duration (from EquipmentTimeEntries for the job, rate resolved via resolveEquipmentRate from LEM-2). Blank rows: description + qty + rate (from WorkOrderEquipmentItems — no equipmentId, no rate labels). Rates only with viewWageDetails. Subtotal combines both sources.
  • SignaturesDisplay: Grid of signature images with signer name, timestamp, GPS.

5. PDF Download Template

Uses the existing ObjectDownloadTemplate model linked to WorkOrder DAAs. Each section (labor, material, equipment, signatures) is toggleable in the template config. Standard DAA fields render as key-value pairs; custom sections get custom PDF renderers.

DAA SlugPDF Rendering
wo-labor-itemsCombined table: timesheet entries (Employee | Trade Level | Hours | Rate) + blank lines (Description | Workers | Hours | Rate). Rates if permitted. Subtotal from both sources.
wo-material-itemsTable: Material | Qty | UoM | Unit Cost | Total. Subtotal.
wo-equipment-itemsCombined table: logged equipment (Equipment | Rate Type | Duration | Rate) + blank lines (Description | Qty | Rate). Rates if permitted. Subtotal from both sources.
wo-signaturesSignature images with signer name, date/time, GPS.
Existing infrastructure: /api/pdf-export + @useanzen/pdf-worker already handle Job PDFs. WO PDFs extend with custom LEM section renderers. ObjectDownloadTemplate seeded for Work Orders in the migration. Admins configure which fields/sections appear via the Property Settings Download Template tab.

8. Acceptance Criteria

  1. 4 tabs: Overview, Invoices, Files, Settings
  2. Summary cards show Total Value, Labor Hours, Material Cost, Equipment, Status, Date of Work
  3. Total Value and rates visible only with viewWageDetails
  4. DataModelDetails renders standard WorkOrder fields
  5. LaborItemsDisplay shows combined read-only view: timesheet entries (from TimeSheetDetailRecords) grouped by employee/date + blank lines (from WorkOrderLaborItems)
  6. EquipmentItemsDisplay shows combined read-only view: logged equipment (from EquipmentTimeEntries) grouped by equipment/date + blank lines (from WorkOrderEquipmentItems)
  7. Material and Signatures rendered as custom display components within Overview
  8. Invoices tab shows read-only invoice list (no create button)
  9. Edit handled by DataModelDetails built-in edit functionality (not a separate header button)
  10. Print PDF generates from download template with toggleable LEM sections
  11. PDF custom renderers for labor/material/equipment tables and signatures
  12. Files tab uses existing Job files
  13. Settings tab scoped to subModel="WorkOrder"
