1. Business Case ▼

Appello currently has no portfolio-wide view of T&M work orders. Each work order lives buried inside an individual job page, making it impossible for office staff to see the full picture — which WOs are pending approval, which are paid, and where labor hours are concentrated.

Clearstory's T&M Log provides exactly this: a labor hours matrix broken down by status and hour type (ST/OT/DT), plus a status overview showing dollar totals by status. This is the operational command center that project managers and billing coordinators need.

Customer Evidence

"We need to be able to quickly see which work orders are paid and which are still pending. Right now I have to click into each job individually." — Rival Insulation
"Is there a way to export the change order log? We need to pull that data into our accounting system." — Vanos Insulation

Competitive Landscape

HAS THIS Clearstory T&M Log with labor hours matrix, status overview, and full filtering. Their signature feature.
HAS THIS Procore T&M Log with configurable columns and export. Part of their project financials suite.

2. Scope ▼

In Scope

  • IN New page at /work-orders accessible from sidebar navigation
  • IN ListTableViews component with work order filter support
  • IN Labor Hours Summary matrix — rows: statuses (Draft, Pending, Approved, Invoiced, Paid), columns: ST / OT / DT / Total
  • IN Status Overview panel — colored status dots with $ amounts and WO counts per status
  • IN Filters: project, company, status, date range
  • IN CSV export of filtered work order data
  • IN "+ New Work Order" button launching create flow

Out of Scope

  • OUT Kanban board view (Phase 2)
  • OUT Email notifications for status changes

4. Data Model ▼

No new Prisma models are required. This feature consumes existing work order data through a new SQL view and a new GraphQL summary type.

WorkOrderListFeed SQL View

A new database view that joins work order jobs with their line items, companies, and projects for efficient list rendering.

CREATE OR REPLACE VIEW WorkOrderListFeed AS
SELECT
    j.id AS jobId,
    j.name AS description,
    j.jobNumber AS woNumber,
    j.status,
    j.createdAt,
    p.id AS projectId,
    p.name AS projectName,
    c.id AS companyId,
    c.name AS companyName,
    j.instanceId,
    COALESCE(SUM(CASE WHEN wli.hourType = 'ST' THEN wli.hours ELSE 0 END), 0) AS stHours,
    COALESCE(SUM(CASE WHEN wli.hourType = 'OT' THEN wli.hours ELSE 0 END), 0) AS otHours,
    COALESCE(SUM(CASE WHEN wli.hourType = 'DT' THEN wli.hours ELSE 0 END), 0) AS dtHours,
    COALESCE(SUM(wli.total), 0) AS grandTotal
FROM Job j
LEFT JOIN Project p ON j.projectId = p.id
LEFT JOIN Company c ON j.companyId = c.id
LEFT JOIN WorkOrderLineItem wli ON wli.jobId = j.id
WHERE j.jobType = 'WORK_ORDER'
  AND j.isDeleted = 0
  AND j.isArchived = 0
GROUP BY j.id;

5. API Surface ▼

Query: workOrderLogSummary

query WorkOrderLogSummary(
    $projectId: String
    $companyId: String
    $status: String
    $dateFrom: DateTime
    $dateTo: DateTime
) {
    workOrderLogSummary(
        projectId: $projectId
        companyId: $companyId
        status: $status
        dateFrom: $dateFrom
        dateTo: $dateTo
    ) {
        laborHours {
            status
            stHours
            otHours
            dtHours
            totalHours
        }
        statusCounts {
            status
            count
            totalValue
        }
        totalCount
    }
}

The work order list itself is served by TableFeed using the WorkOrderListFeed view, leveraging existing ListTableViews infrastructure.

7. Acceptance Criteria ▼

  • User can navigate to /work-orders from the sidebar and see a full list of all work orders across all jobs.
  • Labor Hours Summary matrix displays correct ST/OT/DT totals grouped by work order status.
  • Status Overview panel shows correct dollar totals and WO counts for each status (Draft, Pending, Approved, Invoiced, Paid).
  • User can filter the work order list by project, company, status, and date range.
  • User can export the current filtered view as a CSV file.
  • "+ New Work Order" button opens the work order creation flow.
  • Page respects user permissions — users with viewOnlyMine see only their work orders.
  • Loading, error, and empty states are properly handled.
