1. Business Case ▼

Change orders are currently only visible within individual job pages. There is no portfolio-wide view to see all change orders across all projects — making it impossible to track outstanding revenue, aging COs, or overall contract adjustments at a glance.

Clearstory's COR Log is their signature feature, offering a contract summary with original/approved/pending/in-review/void breakdowns, aging reports, and full export. This is table stakes for specialty contractors managing dozens of active COs.

Customer Evidence

"Is there a way to export the change order log to PDF? We need to pull that data for our monthly reporting." — Vanos Insulation (Q21030-8245)
"We need to track what's paid and what's outstanding across all our projects. Right now there's no single place to see that." — Rival Insulation

2. Scope ▼

In Scope

  • IN New page at /change-order-log accessible from sidebar navigation
  • IN Contract Summary panel — Original Contract / Approved COs / Pending COs / In Review / Void values
  • IN Status Summary — color-coded dollar totals by status (Won, Pending, Lost, Draft)
  • IN CO Log Overview metrics — total count, average days in review, average CO value
  • IN ListTableViews table with CO data (CO#, Title, Date Submitted, Requested Amount, Company, Project, Status, Days in Review)
  • IN Filters: project, company, status, date range
  • IN CSV and PDF export of filtered data

Out of Scope

  • OUT Sent/Received tabs (requires multi-tenant CO model)
  • OUT Ball-in-Court historical analytics (Phase 2)

4. Data Model ▼

ChangeOrderLogFeed SQL View

A new database view joining EstimateChangeOrderRecord with projects, companies, and jobs for efficient list rendering and aggregation.

CREATE OR REPLACE VIEW ChangeOrderLogFeed AS
SELECT
    eco.id,
    eco.estimateNumber AS coNumber,
    eco.name AS title,
    eco.createdAt AS dateSubmitted,
    eco.totalPrice AS requestedAmount,
    eco.status,
    eco.statusChangedAt,
    DATEDIFF(NOW(), eco.createdAt) AS daysInReview,
    p.id AS projectId,
    p.name AS projectName,
    c.id AS companyId,
    c.name AS companyName,
    j.id AS jobId,
    j.name AS jobName,
    eco.instanceId
FROM EstimateChangeOrderRecord eco
LEFT JOIN Job j ON eco.jobId = j.id
LEFT JOIN Project p ON j.projectId = p.id
LEFT JOIN Company c ON j.companyId = c.id
WHERE eco.isDeleted = 0
  AND eco.isArchived = 0
  AND eco.estimateType = 'CHANGE_ORDER';

Schema Change: statusChangedAt

Add a statusChangedAt column to EstimateChangeOrderRecord to track when the status last changed, enabling accurate "Days in Review" calculations.

ALTER TABLE `EstimateChangeOrderRecord`
ADD COLUMN `statusChangedAt` DATETIME(3) NULL;

5. API Surface ▼

Query: changeOrderLogSummary

query ChangeOrderLogSummary(
    $projectId: String
    $companyId: String
    $status: String
    $dateFrom: DateTime
    $dateTo: DateTime
) {
    changeOrderLogSummary(
        projectId: $projectId
        companyId: $companyId
        status: $status
        dateFrom: $dateFrom
        dateTo: $dateTo
    ) {
        contractSummary {
            originalContract
            approvedCOs
            pendingCOs
            inReview
            voidCOs
        }
        statusCounts {
            status
            count
            totalValue
        }
        totalCount
        avgDaysInReview
    }
}

The change order list is served by TableFeed using the ChangeOrderLogFeed view, leveraging existing ListTableViews infrastructure.

7. Acceptance Criteria ▼

  • User can navigate to /change-order-log from the sidebar and see all change orders across all jobs.
  • Contract Summary panel displays correct original contract, approved, pending, in-review, and void values with revised contract total.
  • Status Summary shows color-coded dollar totals for Won, Pending, Lost, and Draft statuses.
  • CO Log Overview displays accurate total count, average days in review, and average CO value.
  • User can filter the change order list by project, company, status, and date range.
  • User can export the current filtered view as CSV and PDF files.
  • "Days in Review" column shows correct aging based on statusChangedAt or createdAt.
  • Page respects user permissions — users with viewOnlyMine see only their change orders.
  • Loading, error, and empty states are properly handled.
