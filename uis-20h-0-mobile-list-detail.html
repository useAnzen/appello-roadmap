WO-6 Work Orders Draft

Mobile WO List & Detail

Work Order list and detail pages on mobile using existing TableDataList and DetailOverview + TabGroup components. Mirrors the RFI list pattern and the Job detail tab pattern. Provides the entry point for creating and viewing work orders on mobile.

Estimate: 1-2 days Requires: WO-1 (JobClassification + sidebar), WO-3d (TableFeed configureColumns)

1. Approach

The mobile app already has well-established patterns for list and detail pages. This feature drops Work Orders into those patterns with minimal custom code:

  • List page: TableDataList with modelName="Job" and defaultWhere filtering to WorkOrder classification. Same pattern as the RFI list page.
  • Detail page: TabGroup with Overview (DetailOverview), Notes, and Files tabs. Same pattern as the Job detail page but scoped to WO-relevant tabs.
No new components needed. TableDataList, DetailOverview, TabGroup, JobNotes, and JobFiles are all existing components. This is purely wiring them up for the WorkOrder subModel with appropriate columns and configuration.

2. Work Order List Page

Route

apps/mobile/src/pages/job-sites/[id]/work-orders/index.tsx

Configuration

<TableDataList<IWOTableData>
    modelName="Job"
    columns={["fullJobNumber", "name", "jobStatusId", "dateOfWork", "projectId"]}
    orderBy={{ column: "createdAt", direction: "DESC" }}
    defaultWhere={{ jobClassification: "WorkOrder" }}
    filterableColumns={["jobStatusId", "projectId"]}
    displayCard={(record) => <WOListCard record={record} />}
    onCreateClick={() => openFlyout(<DataModelForm subModel="WorkOrder" steppedForm ... />)}
    createLabel="Create Work Order"
/>

WOListCard

A simple card component showing WO number, name, status badge, date of work, and project name. Tapping navigates to the detail page.

3. Work Order Detail Page

Route

apps/mobile/src/pages/job-sites/[id]/work-orders/[woId]/index.tsx

Tab structure

Tab Component Notes
Overview DetailOverview modelName="Job", viewSectionType="Details". Shows all WO DAA fields in sectioned read-only view. customColumnRender for line item display (labor, materials, equipment) and signatures.
Notes JobNotes Reuses the existing Job notes component. Notes are polymorphic on Job already. Shows note count badge.
Files JobFiles in FilesProvider Reuses the existing Job files component. Assets are polymorphic on Job. Shows file count badge.

Header

Shows WO number + name, status badge, and an edit button that opens the DataModelForm flyout (non-stepped on detail, since editing is section-at-a-time).

4. Mobile Navigation

Work Orders needs an entry point in the mobile app navigation. This depends on how WO-1 configured the sidebar — the mobile app should mirror the desktop sidebar entry.

  • Add "Work Orders" item to the mobile navigation menu (same position as desktop sidebar)
  • Links to /job-sites/[id]/work-orders
  • Uses the same permission check as the desktop sidebar entry

6. Acceptance Criteria

  1. "Work Orders" item in mobile navigation menu
  2. WO list page shows only WorkOrder-classified jobs
  3. List cards show WO number, name, status badge, date of work, project name
  4. Search filters work orders by name and WO number
  5. Status and project facet filters work
  6. "+" button opens the DataModelForm for WorkOrder creation
  7. Tapping a card navigates to WO detail page
  8. Detail page shows Overview, Notes, and Files tabs
  9. Overview tab renders DetailOverview with WO DAA fields in sectioned layout
  10. Line items (labor, materials, equipment) display via customColumnRender
  11. Signatures display in the Overview tab
  12. Notes tab reuses JobNotes with note count badge
  13. Files tab reuses JobFiles with file count badge
  14. Edit button opens DataModelForm for the existing WO record
