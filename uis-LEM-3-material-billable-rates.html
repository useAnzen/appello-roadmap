Material Billable Rates

Three-tier pricing cascade (Group → Material → Job) that auto-calculates what to charge the customer for materials. Completes the LEM triad alongside labor billable rates (LEM-1) and equipment billable rates (LEM-2).

1. Business Case

Labor has cost vs. billable separation (LEM-1 adds billableValue to TradeLevelRate). Equipment has cost vs. billable separation (LEM-2 adds equipment billable rates). Materials have no equivalent. Today, the only material prices in the system are cost prices — what the contractor pays distributors. There is no mechanism to define what the contractor charges the customer for those materials.

Without material charge-out rates, Work Order material line items either require manual price entry on every line (error-prone, inconsistent) or pass through at cost (zero margin). Neither is acceptable for a production system.

"In construction, they call it labor equipment material, LEM. Right now, we don’t have structured data for labor rates. We don’t have structured data for material rates or equipment rates. It’s like we kind of have to add all three of these for work orders to be functional." — Corey Shelson, March 30, 2026
"What we have right now are price books, which are the prices we pay for the materials when we buy them from our suppliers. Nowhere have we actually set what our margins are on those products or our charge-out prices." — Corey Shelson, April 8, 2026

Design Inspiration: Shopify

Shopify gives every product two fields: Price (what the customer pays) and Cost per item (what you paid). Margin is computed and shown inline. For B2B, Shopify Price Lists apply a percentage rule across the catalog with per-product overrides. We adapt this pattern for construction: a default markup at the material group level, with overrides at the material and job level.

3. Data Model Changes

Three-Tier Pricing Cascade

Tier 3 — Job Override (Highest Priority)

Job.materialMarkupPercent — applies to all materials on this job. For tight-margin projects or preferred customers.

Tier 2 — Material Override

Material.sellPriceOverride (fixed $) OR Material.markupPercentOverride (%) — for specific products with non-standard margins.

Tier 1 — MaterialGroup Default

MaterialGroup.defaultMarkupPercent — set once per category (e.g., “Insulation: 25%”). Covers all materials in the group.

Fallback — Pass-Through

If no markup is configured at any tier, the sell price equals the cost price (zero margin).

Migration SQL

-- Tier 1: MaterialGroup default markup
ALTER TABLE `MaterialGroup` ADD COLUMN `defaultMarkupPercent` DECIMAL(5, 2) NULL;

-- Tier 2: Material-level overrides
ALTER TABLE `Material` ADD COLUMN `sellPriceOverride` DECIMAL(10, 2) NULL;
ALTER TABLE `Material` ADD COLUMN `markupPercentOverride` DECIMAL(5, 2) NULL;

-- Tier 3: Job-level material markup
ALTER TABLE `Job` ADD COLUMN `materialMarkupPercent` DECIMAL(5, 2) NULL;
Why Decimal(5,2)? Supports markup percentages from 0.01% to 999.99%. Most construction markups range from 5% to 50%, but some specialty items can have higher margins. Matches the precision used for discountPercent on DistributorPrice.

4. Resolution Engine

Cost Basis Resolution

The cost basis is the “best available cost” for a material, resolved in this order:

  1. DistributorPrice.effectivePrice (preferred distributor, after discounts) — available when MAT-0 + MAT-9 are shipped in 1.34
  2. MaterialAttributePrice.value (dimension-based catalog price) — available today in the existing system
  3. Manual entry — if no catalog price exists, the user enters cost manually

Sell Price Resolution

function resolveMaterialSellPrice(
    material: Material,
    materialGroup: MaterialGroup,
    job: Job | null,
    costPrice: Decimal
): { sellPrice: Decimal; source: string } {

    // Tier 3: Job override (highest priority)
    if (job?.materialMarkupPercent != null) {
        return {
            sellPrice: costPrice.mul(1 + job.materialMarkupPercent / 100),
            source: "job"
        };
    }

    // Tier 2a: Material fixed sell price
    if (material.sellPriceOverride != null) {
        return {
            sellPrice: material.sellPriceOverride,
            source: "material-fixed"
        };
    }

    // Tier 2b: Material markup %
    if (material.markupPercentOverride != null) {
        return {
            sellPrice: costPrice.mul(1 + material.markupPercentOverride / 100),
            source: "material-percent"
        };
    }

    // Tier 1: MaterialGroup default
    if (materialGroup.defaultMarkupPercent != null) {
        return {
            sellPrice: costPrice.mul(1 + materialGroup.defaultMarkupPercent / 100),
            source: "group"
        };
    }

    // Fallback: pass-through at cost
    return { sellPrice: costPrice, source: "cost" };
}
Pattern precedent: This cascade mirrors the equipment rate resolution in LEM-2 (category default → equipment override → job override). Emma already knows the pattern.

6. UI Specification

6a. Material Detail Page — Pricing Card

Add a “Pricing” card to the Material detail page showing cost, sell price, and margin. The user selects their pricing strategy via radio buttons.

  • Best Cost (read-only): resolved from DistributorPrice or MaterialAttributePrice
  • Sell Price (editable or computed): depends on the selected pricing mode
  • Margin (computed): dollar amount and percentage, shown inline

Pricing mode radio buttons:

  • Use group default — inherits MaterialGroup.defaultMarkupPercent
  • Fixed sell price — user enters sellPriceOverride
  • Custom markup % — user enters markupPercentOverride

6b. MaterialGroup Form — Default Markup

Add a Default Markup % field to the MaterialGroup add/edit form. This sets the Tier 1 default for all materials in the group.

6c. Job Settings — Material Markup Override

Add a Material Markup % field to the Job detail page (likely in the Financial Settings or Pricing section). This provides the Tier 3 job-level override.

6d. WO Material Line Item Editor (Downstream)

When a user selects a Material from the catalog on a WO line item:

  • unitPrice (sell price) auto-populates from resolveMaterialSellPrice()
  • costPrice (cost) auto-populates from the cost basis
  • Both fields are editable — the user can override the auto-populated values
  • Margin is shown inline if the user has viewWageDetails permission

9. Acceptance Criteria

LEM-3a (Schema + Service)

  • Migration adds 4 nullable columns to MaterialGroup, Material, and Job tables without breaking existing data
  • resolveMaterialSellPrice(materialId, jobId?) correctly walks the cascade: Job > Material fixed > Material % > Group % > cost pass-through
  • bestAvailableCost(materialId) returns the lowest available cost from distributor prices or attribute prices
  • The sellPrice GraphQL computed field on Material returns the correctly resolved sell price
  • costBasis, marginPercent, and pricingSource computed fields return correct values
  • If no markup is configured at any tier, the sell price equals the cost price (zero margin, not an error)
  • EditMaterialGroupInput, EditMaterialInput, and EditJobInput accept the new pricing fields

LEM-3b (UI)

  • The Material detail page shows a Pricing card with cost, sell price, and margin ($ and %)
  • The Material detail page pricing mode radio buttons correctly switch between group default, fixed sell price, and custom markup %
  • Changing the pricing mode and saving persists the correct fields (sellPriceOverride or markupPercentOverride)
  • The MaterialGroup add/edit form includes a Default Markup % field
  • The Job detail page includes a Material Markup % field in the financial/pricing section

Downstream (validated by WO-2 and MAT-2a)

  • When a Material is selected on a WO line item, unitPrice auto-populates from the resolved sell price (WO-2)
  • The LEM combined report shows materialBillableValue alongside materialCost (MAT-2a)
