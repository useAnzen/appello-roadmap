1 · Business Case & Customer Evidence ▼

The “M” in LEM. Labor tracking is covered by timesheets, Equipment tracking is covered by UIS-15H — but Material consumption (“we used 24 batts of R-30 today”) has no native tracking. Customers currently use custom forms as a workaround, losing structured data, cost-code allocation, and any connection to job financials.

Customer Evidence

“A full material management system will provide a more robust solution than our current form-based approach.” — St. Clair Mechanical (Demo, Feb 2026)
“We need a T&M daily report with separate sections for labor, materials, and equipment.” — AB Insulation (Form-Building Session, Nov 2024)

Value Proposition

⚡
Complete LEM Trifecta
Labor, Equipment, and Material all tracked natively — no more form workarounds for the “M” in LEM.
📈
Daily Burn Rate Visibility
See exactly what materials are consumed per day per job, enabling early detection of overruns.
💰
P&L Accuracy
Material costs flow into the Job Financial Summary — computed downstream from catalog or distributor pricing.
📌
Cost Code Allocation
Every material consumption entry can be tagged to a cost code for granular budget tracking.

Competitive Landscape

HAS Procore T&M Tickets with material line items and daily logs
HAS Clearstory T&M Tags with material lines on daily reports
HAS ServiceTitan Material tracking per job with inventory integration
PARTIAL Appello Custom forms workaround — no structured data, no P&L integration

2 · Scope ▼

In Scope

  • IN MaterialConsumptionEntry Prisma model (no stored cost — computed downstream)
  • IN Desktop entry UI on Job detail page (form + table of existing entries)
  • IN LEM combined report tab joining labor + equipment + material data
  • IN LEMReportView SQL view aggregating all three LEM dimensions
  • IN PDF export of LEM report
  • IN Material consumption costs in Job Financial Summary (computed downstream from Material catalog or DistributorPrice)
  • IN Draft / Submitted / Approved workflow status on consumption entries

Out of Scope

  • OUT Mobile entry (deferred to MAT-2b)
  • OUT Inventory deduction on consumption
  • OUT Offline support
  • OUT Bulk import of consumption data

4 · Data Model ▼

MaterialConsumptionEntry

Design decision: No unitCost or total fields stored. Cost is computed downstream from the Material catalog price or DistributorPrice at query time. This avoids stale pricing and keeps the consumption entry purely volumetric. 
model MaterialConsumptionEntry {
  id            String    @id @db.VarChar(21)
  isDeleted     Boolean   @default(false)
  isArchived    Boolean   @default(false)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @default(now()) @updatedAt
  jobId         String    @db.VarChar(21)
  Job           Job       @relation(fields: [jobId], references: [id])
  materialId    String?   @db.VarChar(21)
  Material      Material? @relation(fields: [materialId], references: [id])
  dateOfWork    DateTime
  quantity      Decimal   @db.Decimal(10, 2)
  unitOfMeasure String?   @db.VarChar(50)
  costCodeId    String?   @db.VarChar(21)
  notes         String?   @db.Text
  submittedByUserId String?   @db.VarChar(21)
  SubmittedBy   User?     @relation(fields: [submittedByUserId], references: [id])
  status        String    @default("Draft") @db.VarChar(50)
  @@index([jobId, dateOfWork, isDeleted])
  @@index([materialId, isDeleted])
}

Field Reference

FieldTypeNotes
materialIdFK → MaterialOptional — allows freetext description when no catalog entry exists
dateOfWorkDateTimeThe date the material was consumed (not createdAt)
quantityDecimal(10,2)Required. Must be > 0
unitOfMeasureVarChar(50)e.g. “batt”, “ft”, “ea”, “gal” — auto-populated from Material if linked
costCodeIdFK → CostCodeOptional cost code allocation
submittedByIdFK → UserWho logged the consumption
statusVarChar(50)“Draft” | “Submitted” | “Approved”

LEMReportView (SQL View)

-- apps/api/prisma/views/LEMReportView.sql
-- Joins labor (timesheets), equipment (equipment logs), and material consumption
-- Aggregated by jobId + dateOfWork
SELECT
  jobId,
  dateOfWork,
  SUM(laborHours)     AS laborHours,
  SUM(laborCost)      AS laborCost,
  SUM(equipmentHours) AS equipmentHours,
  SUM(equipmentCost)  AS equipmentCost,
  SUM(materialQty)           AS materialQty,
  SUM(materialCost)          AS materialCost,
  SUM(materialBillableValue) AS materialBillableValue,
  SUM(laborCost) + SUM(equipmentCost) + SUM(materialCost) AS dailyCostTotal,
  SUM(laborBillable) + SUM(equipmentBillable) + SUM(materialBillableValue) AS dailyBillableTotal
FROM (
  -- Labor CTE from timesheets + TradeLevelRate.billableValue (LEM-1)
  -- Equipment CTE from equipment logs + equipment billable rates (LEM-2)
  -- Material CTE from MaterialConsumptionEntry + resolveMaterialSellPrice (LEM-3)
) combined
GROUP BY jobId, dateOfWork
ORDER BY dateOfWork DESC;

7 · Acceptance Criteria ▼

  • A user with materialorder.create permission can log a material consumption entry from the Job detail page
  • The entry form requires dateOfWork and quantity > 0; submission is blocked if either is missing
  • Either materialId (catalog picker) or a freetext description must be provided — both empty is rejected
  • When a Material is selected from the catalog, unitOfMeasure auto-populates from the Material record
  • New entries default to “Draft” status
  • Draft entries can be edited or deleted by the submitter; Submitted entries can only be edited by approvers
  • The status workflow supports Draft → Submitted → Approved transitions; no backward transitions from Approved
  • The bulk approveMaterialConsumptionEntries mutation transitions multiple entries to Approved in one call
  • No unitCost or total fields are stored on MaterialConsumptionEntry — costs and billable values are computed downstream
  • The LEM Report tab displays a combined daily view joining labor (timesheets), equipment (equipment logs), and material consumption with both cost and billable values
  • The LEM Report shows materialBillableValue (from LEM-3 cascade) alongside materialCost for materials, matching the cost vs. billable pattern used for labor and equipment
  • The LEM Report supports date range filtering and shows a summary totals row
  • The LEM Report can be exported as a formatted PDF with job header, daily breakdown, and signature line
  • Material consumption costs appear in the Job Financial Summary, computed from Material catalog price or DistributorPrice at query time
  • Cost code allocation on consumption entries flows through to cost-code-level budget reports
