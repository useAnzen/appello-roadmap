Itemized Estimate → Simple Invoice

createInvoiceFromEstimate(estimateChangeOrderRecordId) creates a JobBill with template: Simple and one JobBillLineItem per EstimateLineItem. Optional sourceEstimateChangeOrderRecordId on JobBill for traceability. Confirmation modal before create (same UX pattern as WO-7).

1. Business Case ▼

Eliminates re-keying from winning estimate into billing. Clement and similar residential workflows need estimate-to-invoice without a progress report.

2. Scope ▼

In Scope

  • IN Mutation + model method on JobBill or EstimateChangeOrderRecord
  • IN Button on estimate detail when type = Estimate, creationType = Itemized, status = Won
  • IN Confirmation modal: line count, grand total, job selection (default estimate job)

Out of Scope

  • OUT Progress-based invoice generation (existing flow)
  • OUT Automatic tax jurisdiction resolution beyond existing job bill tax UX

3. UI Concepts

Primary action on EstimateChangeOrderDetails (header or Billing area). Modal pattern mirrors WO-7 ConversionConfirmationModal — summary first, destructive/irreversible action clearly labeled.

3.1 Generate invoice in full detail header

Place Generate invoice alongside existing estimate actions (e.g. Export PDF, Duplicate, status controls) in EstimateChangeOrderDetails header / toolbar — not buried in Billing tab only.

3.2 Post-success

After successful createInvoiceFromEstimate: close modal; show success toast; navigate to JobBill detail (or job Billing tab with bill selected) per existing billing deep-link patterns. User should land on a view showing mapped line items.

Invoice INV-9081 created. 6 lines · $7,606.54 · Open invoice

3.3 Error state

On GraphQL / network failure: keep modal open (or return to detail); show inline error in modal footer + Retry button. Do not clear line selection. Log server message for support.

Create Simple invoice from estimate?

Request failed: insufficient billing permission. [ Retry ]
Close

3.4 Duplicate invoice guard (optional product)

If PM blocks a second invoice from the same estimate: render primary button as disabled, tooltip “An invoice already exists for this estimate.”, and text link View existing invoice (opens JobBill). If PM allows duplicates, show secondary confirm in modal instead.

7. Acceptance Criteria ▼

  • Button only visible for Won itemized estimates with at least one line
  • Created bill total matches sum of line prices (+ taxes per existing Simple rules)
  • Navigates to bill detail after success
  • Source estimate ID stored on JobBill when column exists
