1. Business Case ▼

Estimators need the same mental model as field T&M capture: pick catalog rows or type free-form descriptions, see unit cost populate, adjust quantity and markup.

Combination of dropdown picks and manual dollar entry on the same grid. — Clement / product direction (April 2026)

2. Scope ▼

In Scope

  • IN EstimateLineItemEditor.tsx — table + add row + delete row + row order
  • IN TableFeed or dedicated query for labor (user + trade), material, equipment pick lists with search
  • IN Client calls resolveEstimateLineItemUnitCost on selection to seed unitCost and unit
  • IN Wire create/edit/delete to GraphQL from IE-1 (per-row or batched when IE-3 lands)
  • IN Show editor when creationType = Itemized

Out of Scope

  • OUT Record-level overhead rollups (IE-3)
  • OUT PDF / invoice (IE-4, IE-5)

3.2 Per-type add flows (after pick)

Labor Labor
1. Open Add line → Labor group.
2. useGetTableDataRecordSetQuery (User) → pick worker.
3. Show primary UserTradeLevel; unit + unitCost from TradeLevelRate.billableValue via resolver.
4. Row inserted; qty editable.
Material Mat
1. Search material catalog.
2. On select, resolver fills unit + unitCost (MaterialAttributePrice / LEM-3 when present).
3. Description = material display name + optional note.
Equipment Eq
1. Pick equipment record.
2. Billing unit selector: hr or day (per product).
3. resolveEquipmentRate seeds cost.
4. Row shows equipment name + unit.
Custom Custom
1. Choose Custom → blank row.
2. All FKs null; user types description, qty, unit, unit cost manually.
3. No resolver call until optional “Suggest rate” (out of v1 unless PM adds).

5. Data & API ▼

Prisma model, CRUD, and resolveEstimateLineItemUnitCost are defined in IE-1. IE-2 consumes estimateLineItems(estimateChangeOrderRecordId) and row mutations (or batched upsert once IE-3 extends the parent mutation).

Optional until IE-3: upsertEstimateLineItems(estimateId, items[]) for transactional save with header totals.

7. Acceptance Criteria ▼

  • User can add, edit, remove rows without full page reload
  • Selecting a catalog row fills unit cost via server resolve
  • Labor row ties to a specific user when product requires user-first labor
  • Custom row allows manual description and costs with null catalog FKs
  • Total cost, markup fields, and line price stay numerically consistent
  • Permissions match existing estimate edit rules
