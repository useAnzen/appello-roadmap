Itemized Estimate PDF

Branch the existing estimate PDF pipeline when creationType = Itemized: render EstimateLineItem rows in a client-facing table. Support “display to client” toggles for unit cost, markup, and internal notes (Buildertrend-style).

1. Business Case ▼

Customers expect a printable quote that matches the on-screen itemized grid. Internal PDFs may show cost and markup; customer PDFs often hide margin columns.

2. Scope ▼

In Scope

  • IN PDF branch for Itemized in EstimateChangeOrderPDF.tsx (or sibling component)
  • IN Line table: Description, Qty, Unit, Unit Price, Line Total (configurable)
  • IN Instance or per-estimate flags for column visibility
  • IN Wire pdf-export.ts case to pass itemized flag + column options

Out of Scope

  • OUT Invoice PDF (existing Job bill PDFs)

3.2 Multi-page behavior

When line items overflow the printable region: repeat table header on each page; totals and signature block appear on the last page only (or repeat subtotal per page — TBD). PDF engine (@react-pdf or current pipeline) must support page breaks on row boundaries.

4. PDF display settings (UI) ▼

Buildertrend-style toggles for what appears on the client PDF. Internal PDF can default to full column set.

Export / template options

TBD (product): Persist toggles on instance defaults, per-estimate override, or PDF modal only — spec UI stands regardless.

4.1 Where settings live (placement options)

Engineering picks one primary surface; document in implementation PR.

A. Export PDF modal / flyout — User chooses Client vs Internal + column toggles each time they export. Best when sales wants per-send flexibility.
B. Estimate record settings — Toggles stored on EstimateChangeOrderRecord (or JSON options column); PDF uses last saved values.
C. Instance defaults — Admin sets defaults; estimate can override (combines A+B).

6. Acceptance Criteria ▼

  • Itemized estimates generate PDF with correct line count and totals
  • Client mode hides cost/markup columns per configuration
  • Full / BidFirst estimates still use existing worksheet PDF (no regression)
