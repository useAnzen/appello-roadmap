  • MaterialGroup add/edit form has a Default Markup % field.

Removed from earlier draft: the “Materials on Job” / “Pricing Cascade Preview” mockup. The data model has no concept of materials curated to a specific job — the job-level markup applies at line-item time when a material is priced against the job, not from a job-scoped material list. Don’t reintroduce that view.

1.2 Data fetch

Use a colocated gql query at the bottom of the file:

const GET_MATERIAL_PRICING = gql`
    query GetMaterialPricing($id: ID!) {
        material(id: $id) {
            id
            sellPriceOverride
            markupPercentOverride
            costBasis
            sellPrice
            marginPercent
            pricingSource
            MaterialGroup {
                id
                name
                defaultMarkupPercent
            }
        }
    }
`;

For mutation use the existing editMaterial. After codegen runs, the typed hook should be useEditMaterialMutation (mirrors useEditMaterialGroupMutation in MaterialGroupAddEditForm.tsx). If the hook isn’t generated yet, write the mutation by hand:

const EDIT_MATERIAL_PRICING = gql`
    mutation EditMaterialPricing($data: EditMaterial!) {
        editMaterial(data: $data) {
            id
            sellPriceOverride
            markupPercentOverride
            sellPrice
            marginPercent
            pricingSource
        }
    }
`;

1.4 Pricing-mode logic

Selected radioOn save
Use group defaultsellPriceOverride: null, markupPercentOverride: null
Fixed sell pricesellPriceOverride: <value>, markupPercentOverride: null
Custom markup %sellPriceOverride: null, markupPercentOverride: <value>

Initial radio state derives from the loaded data:

  • sellPriceOverride != null → Fixed sell price.
  • markupPercentOverride != null → Custom markup %.
  • Otherwise → Use group default.

If the group has no defaultMarkupPercent set, surface a hint under the radio (“Group has no default — pass-through at cost”).

1.5 Save UX

  • Use useNotification() to show a success toast on save (consistent with MaterialGroupAddEditForm.tsx).
  • Refetch on save so sellPrice / marginPercent / pricingSource reflect the new state. Apollo’s refetchQueries: [GET_MATERIAL_PRICING] is the simplest path.
  • Disable Save while loading.

1.6 Skeleton

import { FC, useEffect, useState } from "react";
import { gql, useQuery, useMutation } from "@apollo/client";
import { ClassNames } from "@useanzen/shared-utils";
import { Card, Button, SaveCancelButtons } from "@useanzen/ui";

import { FormControl, FormStack } from "~root/components/_FormControls";
import { useNotification } from "~root/hooks";

interface IMaterialPricingCardProps {
    materialId: string;
    className?: string | ClassNames;
}

type Mode = "group" | "fixed" | "percent";

const MaterialPricingCard: FC<IMaterialPricingCardProps> = ({
    materialId,
    className = ""
}) => {
    const { data, loading, refetch } = useQuery(GET_MATERIAL_PRICING, {
        variables: { id: materialId }
    });
    const [editMaterial, { loading: saving }] = useMutation(EDIT_MATERIAL_PRICING);
    const { notify } = useNotification();

    const [mode, setMode] = useState<Mode>("group");
    const [fixed, setFixed] = useState<string>("");
    const [percent, setPercent] = useState<string>("");

    useEffect(() => {
        const m = data?.material;
        if (!m) return;
        if (m.sellPriceOverride != null) {
            setMode("fixed");
            setFixed(String(m.sellPriceOverride));
        } else if (m.markupPercentOverride != null) {
            setMode("percent");
            setPercent(String(m.markupPercentOverride));
        } else {
            setMode("group");
        }
    }, [data?.material?.id]);

    const handleSave = async () => {
        const patch = {
            id: materialId,
            sellPriceOverride: mode === "fixed" ? parseFloat(fixed) : null,
            markupPercentOverride: mode === "percent" ? parseFloat(percent) : null
        };
        await editMaterial({ variables: { data: patch } });
        await refetch();
        notify({ type: "success", message: "Pricing updated" });
    };

    // ... render summary row + radio block ...

    return (
        <Card className={new ClassNames().add(className).list()}>
            {/* layout per mockup 3a */}
        </Card>
    );
};

export { MaterialPricingCard };

// gql definitions stay here at the bottom (per house style)

const EDIT_MATERIAL_PRICING = gql`...`;
