1 · Business Case & Customer Evidence ▼

Contractors receive price books from distributors — spreadsheets with thousands of SKUs and prices. Manual entry is unsustainable. M-X Insulation explicitly needs Crossroads price book ingestion.

Customer Evidence

"Ingesting all of the Crossroads price books with all the discounts" — M-X Insulation
"300-page Excel files from our distributors — we can’t type all that in" — Vanos Insulation
📦
Bulk Ingestion
Import thousands of SKUs and prices from a single spreadsheet upload instead of manual entry.
🔗
Column Mapping Wizard
Intelligent auto-detect maps source columns to target fields with manual override capability.
📊
Price Comparison
Side-by-side view of old vs new pricing so operators can see what changed before committing.
📋
Import History
Full audit trail of every import — who imported, when, row counts, and error logs.

Competitive Landscape

  • IN PriceBookImport Prisma model for tracking import history

  • IN 4-step frontend wizard: Upload → Map Columns → Preview → Import
  • IN Price comparison view on Supplier detail page

Out of Scope

  • OUT EDI / API integration with distributors
  • OUT Automatic scheduled imports
  • OUT Export price books

4 · Data Model ▼

PriceBookImport

model PriceBookImport {
  id            String    @id @default(nanoid()) @db.VarChar(21)
  instanceId    String    @db.VarChar(21)
  distributorId String    @db.VarChar(21)
  fileName      String    @db.VarChar(500)
  status        String    @default("Pending") @db.VarChar(50)
  rowCount      Int       @default(0)
  successCount  Int       @default(0)
  errorCount    Int       @default(0)
  errorLog      Json?
  importedById  String    @db.VarChar(21)
  isDeleted     Boolean   @default(false)
  isArchived    Boolean   @default(false)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @default(now()) @updatedAt

  Instance      Instance     @relation(fields: [instanceId], references: [id])
  Distributor   Company      @relation(fields: [distributorId], references: [id])
  ImportedBy    User         @relation(fields: [importedById], references: [id])

  @@index([instanceId])
  @@index([distributorId])
  @@index([importedById])
}

Status values: Pending, Processing, Complete, PartialSuccess, Failed.

7 · Acceptance Criteria ▼

  • User can upload a CSV file and the system parses it into rows and columns
  • User can upload an Excel (.xlsx) file and the system parses it into rows and columns
  • Column mapping auto-detects common field names (SKU, Description, Price, UOM) with ≥80% accuracy
  • User can manually override any auto-detected column mapping
  • Preview step shows total row count, validation error count, and price change count
  • Rows with validation errors are highlighted red with error description
  • Successful rows create ManufacturerProduct + DistributorPrice records
  • Duplicate handling supports both "Update Existing" and "Skip" strategies
  • Price comparison shows old price vs new price for existing products
  • Import history is viewable with row counts, error counts, and timestamps
  • Error log is downloadable after import
  • Permission check: material.create required to initiate import
