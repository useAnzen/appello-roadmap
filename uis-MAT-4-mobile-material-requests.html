1 · Business Case & Customer Evidence ▼

The single most requested mobile capability. Today, material requests happen via text, WhatsApp, and phone calls — untracked, uncosted, and unlinked to jobs.

Customer Evidence

"Amazon-style product database… coming very soon" — SCT Mechanical (Product Demo)
"I have a material management problem. Come solve it." — Matt Vanos, Vanos Insulation (Origin Story)
"The person who gets the request says, this can be fulfilled from inventory, this needs a PO." — Corey Shelson (Nov 2025)
📱
Field-First
Purpose-built mobile form — field workers submit requests in under 30 seconds from the job site.
📦
Catalog + Free-Text
Pick from the material catalog or type in a free-text description for items not yet cataloged.
📸
Photo Attachments
Snap a photo of the product label, spec sheet, or installed item for the office team to reference.
📥
Desktop Inbox
Office team sees all incoming requests in a filterable inbox — acknowledge, assign, and convert to PO.

2 · Scope ▼

In Scope

  • IN MaterialRequest + MaterialRequestItem Prisma models
  • IN Mobile request form: catalog picker + free-text + photo capture
  • IN Desktop Material Request Inbox page with filtering
  • IN One-click acknowledge and assign actions
  • IN Notification on new request (push to office users)
  • IN Request history on Job detail page
  • IN Priority via PropertyValue

Out of Scope

  • OUT Auto-PO generation from requests (future)
  • OUT Inventory fulfillment check
  • OUT Request approval workflow

4 · Data Model ▼

MaterialRequest

model MaterialRequest {
  id            String    @id @default(nanoid()) @db.VarChar(21)
  instanceId    String    @db.VarChar(21)
  isDeleted     Boolean   @default(false)
  isArchived    Boolean   @default(false)
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @default(now()) @updatedAt
  jobId         String    @db.VarChar(21)
  requestedById String    @db.VarChar(21)
  priorityId    String?   @db.VarChar(21)
  status        String    @default("Submitted") @db.VarChar(50)
  notes         String?   @db.Text

  Instance      Instance  @relation(fields: [instanceId], references: [id])
  Job           Job       @relation(fields: [jobId], references: [id])
  RequestedBy   User      @relation(fields: [requestedById], references: [id])
  Priority      PropertyValue? @relation(fields: [priorityId], references: [id])
  Items         MaterialRequestItem[]

  @@index([jobId, isDeleted])
  @@index([instanceId])
  @@index([requestedById])
}

MaterialRequestItem

model MaterialRequestItem {
  id                String    @id @default(nanoid()) @db.VarChar(21)
  isDeleted         Boolean   @default(false)
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @default(now()) @updatedAt
  materialRequestId String    @db.VarChar(21)
  materialId        String?   @db.VarChar(21)
  description       String?   @db.VarChar(500)
  quantity          Decimal   @db.Decimal(10, 2)
  unitOfMeasure     String?   @db.VarChar(50)

  MaterialRequest   MaterialRequest @relation(fields: [materialRequestId], references: [id])
  Material          Material?       @relation(fields: [materialId], references: [id])

  @@index([materialRequestId, isDeleted])
}

Status constants: Submitted, Acknowledged, InProgress, Fulfilled, Cancelled.

7 · Acceptance Criteria ▼

  • Mobile form submits a material request linked to the current Job context
  • Catalog picker searches existing Material records by name or SKU
  • Free-text items are supported when the material is not in the catalog
  • Photos can be attached to the request from the device camera or gallery
  • Desktop inbox shows all material requests with columns: Request #, Job, Requested By, Items, Priority, Status, Date
  • Inbox can be filtered by status, job, and priority
  • Acknowledging a request updates status to Acknowledged and sends a notification to the requester
  • Request detail slideout shows all items, photos, notes, and action buttons
  • Request history is visible on the Job detail page
  • Permission materialorder.create is required to submit a request
  • Office users receive a notification when a new request is submitted
  • Priority is selectable via PropertyValue dropdown (Normal, Urgent, Critical)
