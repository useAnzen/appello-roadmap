Line Item Editors

Custom input components for Labor, Material, and Equipment line items that plug into the DataModelForm via customFormControl. Each supports structured (from rate tables/catalog) and blank (manual entry) lines. Subtotals render within each editor. All state is local until form-level save.

Sequence: WO-3b of 3 Estimate: 4-5 days Requires: WO-3a (DAA positions), WO-2 (line item models)

1. How Editors Plug Into the DataModel Form

WO-3a created 4 DAA positions (wo-labor-items, wo-material-items, wo-equipment-items, wo-signatures) with placeholder <p> tags in customFormControl. WO-3b replaces the first three with real editor components.

Integration pattern (from PR #6831)

const customFormControl = {
    "wo-labor-items": (formData, setFormData) =>
        <LaborItemEditor
            jobId={jobId}
            dateOfWork={formData["wo-date-of-work"]}
            items={laborItems}
            setItems={setLaborItems}
        />,

    "wo-material-items": (formData, setFormData) =>
        <MaterialItemEditor
            jobId={jobId}
            items={materialItems}
            setItems={setMaterialItems}
        />,

    "wo-equipment-items": (formData, setFormData) =>
        <EquipmentItemEditor
            jobId={jobId}
            items={equipmentItems}
            setItems={setEquipmentItems}
        />,

    "wo-signatures": () => <p>Signatures</p>  // WO-3c
};

Key behavior rules

  • Local state only: Editors maintain their own state via items / setItems. Nothing saves to the database until the parent form's Save Draft or Submit is clicked.
  • Subtotals within editors: Each editor renders its own subtotal at the bottom. For labor and equipment structured lines, subtotals are only shown if user has viewWageDetails permission. Material subtotals always shown (sell prices auto-populated from LEM-3 cascade). Cost shown alongside sell price if user has viewWageDetails.
  • On form save: The parent form collects all line item state and calls upsertWorkOrderLineItems (from WO-2) in the same transaction as the Job save.
  • Structured + blank lines: Each editor has an "+ Add" dropdown button with "Structured Line" and "Blank Line" options. Both types coexist in the same list.

2. LaborItemEditor

Component interface

interface ILaborItemEditorProps {
    jobId: string;
    dateOfWork: Date | null;  // determines which EmployeeAgreementContract is active
    items: ILaborRow[];
    setItems: (items: ILaborRow[]) => void;
}

type ILaborRow = IStructuredLaborRow | IBlankLaborRow;

interface IStructuredLaborRow {
    type: "structured";
    orderNo: number;
    employeeAgreementId: string;
    tradeLevelId: string;
    qtyOfWorkers: number;
    rateLabelDurations: Record<string, number>;  // rateLabelId → seconds
}

interface IBlankLaborRow {
    type: "blank";
    orderNo: number;
    description: string;
    qtyOfWorkers: number;
    submittedDuration: number;  // seconds
    rate: number;
}

Structured line behavior

  • Employee Agreement: Auto-selected from the submitting user. If user has multiple agreements, shows a selector. If one agreement, shows read-only with "(auto)".
  • Trade Level: Dropdown populated from the agreement's trade levels. Changing trade level reloads the rate label columns.
  • Rate label columns: Loaded from the active EmployeeAgreementContract for the WO's dateOfWork. Columns are the EmployeeAgreementRateLabel records where isLineItem = false (e.g., Wages 1x, Wages 1.5x, Wages 2x). Hours entered per column. Mirrors the timesheet form pattern from TimesheetRateLabelColumns.tsx.
  • Qty of Workers: Multiplier. Default 1.
  • No rates shown: Hours only on the form. Rates applied at invoice generation (WO-7).
  • Database mapping: Each rate label column with hours > 0 becomes a separate WorkOrderLaborItem record with the same orderNo. The tradeLevelRateId is resolved from (contract + tradeLevel + rateLabel).

Blank line behavior

  • Description (free text, required), Workers qty, Hours (single input), Rate ($/hr, manual entry)
  • No agreement, no trade level, no rate label columns
  • Saves as a single WorkOrderLaborItem with tradeLevelRateId = null and rate set

3. MaterialItemEditor

Component interface

interface IMaterialItemEditorProps {
    jobId: string;
    items: IMaterialRow[];
    setItems: (items: IMaterialRow[]) => void;
}

interface IMaterialRow {
    type: "structured" | "blank";
    orderNo: number;
    materialId?: string;       // set for structured, null for blank
    description?: string;
    quantity?: number;
    unitPrice?: number;        // sell price (auto from LEM-3 cascade, editable)
    costPrice?: number;        // cost basis (read-only for margin visibility)
    total?: number;            // auto-calculated: qty * unitPrice
}

Structured line behavior

  • Material selector: CustomCombobox searching the Material catalog (Solr-indexed). Selecting a material auto-populates description (name) and UoM (from MaterialGroup).
  • Sell Price (unitPrice): Auto-populated from resolveMaterialSellPrice() (LEM-3 cascade: Job > Material > Group > cost pass-through). Editable — user can override the auto-populated value.
  • Cost (costPrice): Auto-populated from cost basis (DistributorPrice or MaterialAttributePrice). Shown read-only for margin visibility when user has viewWageDetails permission.
  • Total: Auto-calculated: quantity × unitPrice. Updated as the user types.

Blank line behavior

  • Description (free text, required), Qty, Unit Cost, Total (auto-calculated)
  • No material catalog lookup
  • Saves as WorkOrderMaterialItem with materialId = null

4. EquipmentItemEditor

Component interface

interface IEquipmentItemEditorProps {
    jobId: string;
    items: IEquipmentRow[];
    setItems: (items: IEquipmentRow[]) => void;
}

type IEquipmentRow = IStructuredEquipmentRow | IBlankEquipmentRow;

interface IStructuredEquipmentRow {
    type: "structured";
    orderNo: number;
    equipmentId: string;
    equipmentRateLabelId: string;
    per: string;
    operatorIncluded: boolean;
    isFullDayBilling: boolean;
    submittedDuration: number;  // seconds
}

interface IBlankEquipmentRow {
    type: "blank";
    orderNo: number;
    description: string;
    quantity: number;
    per?: string;
    unitRate: number;
}

Structured line behavior

  • Equipment selector: CustomCombobox searching Equipment catalog (Solr-indexed).
  • Rate Type: Dropdown populated from EquipmentRateLabel for the selected equipment's category.
  • Per: Dropdown (Hour, Day, Week, Month, Year). May be pre-set from EquipmentAssignment billing context (business question from UIS-15H).
  • Duration: Input labeled based on per selection ("hrs", "days", etc.). Stored in seconds.
  • No rates shown: Rates resolved from LEM-2 at invoice generation.

Blank line behavior

  • Description (free text, required), Qty, Per (unit label), Rate (manual)
  • No equipment catalog lookup, no rate labels
  • Saves as WorkOrderEquipmentItem with equipmentId = null and unitRate set

6. Acceptance Criteria

  1. Labor — Structured: Employee Agreement auto-selects, trade level dropdown loads rate label columns from active contract for dateOfWork
  2. Labor — Structured: Hours entered per rate label column, qty of workers multiplier, no rates shown
  3. Labor — Blank: Free-text description, manual hours + rate + workers qty
  4. Material — Structured: Material catalog selector auto-populates name + UoM, manual unit cost, auto-calculated total
  5. Material — Blank: Free-text description, manual qty + unit cost + total
  6. Equipment — Structured: Equipment selector loads rate labels from category, per dropdown, duration input, no rates shown
  7. Equipment — Blank: Free-text description, manual qty + per + rate
  8. All three editors have "+ Add" dropdown with "Structured Line" and "Blank Line" options
  9. Remove button (×) on each row deletes the line item from local state
  10. Subtotals render at the bottom of each editor (hours for labor/equipment, dollars for materials)
  11. Local state only — no database writes until parent form Save Draft or Submit is clicked
  12. On form save, all line items collected and passed to upsertWorkOrderLineItems mutation (WO-2)
  13. Existing line items loaded from database when editing an existing Work Order
  14. Structured and blank lines can coexist in the same editor
