EstimateLineItem Schema & Model

Introduce EstimateLineItem as the persisted backing store for itemized estimate rows. Nullable catalog FKs support structured picks; custom rows omit FKs. Server-side helpers resolve default unit cost from LEM rate tables.

Estimate: 3–4 days Parent: UIS-IE

1. Business Case ▼

Itemized quoting requires rows that are not constrained to a single AccountItem per line. A dedicated model keeps the legacy GL worksheet intact while enabling catalog-backed pricing.

Structured rows must pull unit cost from rate tables so estimators are not re-keying prices. — Product direction (LEM alignment)

2. Scope ▼

In Scope

  • IN Prisma model EstimateLineItem + enum EstimateLineItemType (Labor, Material, Equipment, Custom)
  • IN Nullable tradeLevelId, materialId, equipmentId; required estimateChangeOrderRecordId
  • IN Fields: description, quantity, unit, unitCost, totalCost, markupPercent, markupDollarValue, price, orderNo, soft delete flags
  • IN Model class + BaseModel list/get/create/edit patterns
  • IN Service resolveEstimateLineItemUnitCost() delegating to TradeLevelRate / Material pricing / EquipmentRateValue

Out of Scope

  • OUT Frontend editor (IE-2)
  • OUT PDF and invoice conversion (IE-4, IE-5)

3. Data Model ▼

enum EstimateLineItemType {
  Labor
  Material
  Equipment
  Custom
}

model EstimateLineItem {
  id                          String   @id @db.VarChar(21)
  instanceId                  String   @db.VarChar(21)
  estimateChangeOrderRecordId String   @db.VarChar(21)
  lineItemType                EstimateLineItemType
  tradeLevelId                String?  @db.VarChar(21)
  materialId                  String?  @db.VarChar(21)
  equipmentId                 String?  @db.VarChar(21)
  userId                      String?  @db.VarChar(21)
  description                 String?  @db.VarChar(500)
  quantity                    Decimal? @db.Decimal(10, 4)
  unit                        String?  @db.VarChar(50)
  unitCost                    Decimal? @db.Decimal(10, 2)
  totalCost                   Decimal? @db.Decimal(10, 2)
  markupPercent               Decimal? @db.Decimal(10, 2)
  markupDollarValue           Decimal? @db.Decimal(10, 2)
  price                       Decimal? @db.Decimal(10, 2)
  orderNo                     Int      @default(0)
  isDeleted                   Boolean  @default(false)
  isArchived                  Boolean  @default(false)
  createdAt                   DateTime @default(now())
  updatedAt                   DateTime @updatedAt
  EstimateChangeOrderRecord   EstimateChangeOrderRecord @relation(...)
  TradeLevel                  TradeLevel?  @relation(...)
  Material                    Material?    @relation(...)
  Equipment                   Equipment?   @relation(...)
  User                        User?        @relation(...)
}

Note: Optional userId on labor rows aligns with the user-first labor rule for WO line items when the same picker is reused in IE-2.

6. Acceptance Criteria ▼

  • Migration creates EstimateLineItem with indexes on estimateChangeOrderRecordId and instanceId
  • CRUD respects BaseModel permissions scoped to estimate permissions
  • resolveEstimateLineItemUnitCost returns billable rate for TradeLevel when LEM-1 data present
  • Equipment path calls existing resolveEquipmentRate (LEM-2)
  • Material path uses Material pricing service when LEM-3 available; otherwise returns null or catalog cell per interim rule
