1 · Business Case & Customer Evidence ▼

The most critical procurement gap. Today “Purchase Order” in Appello means the customer’s PO number — an inbound reference. Contractors need outbound POs to suppliers — formal documents listing materials, quantities, agreed prices, and delivery terms. The November 2025 prototype demonstrated PO creation with an e-commerce metaphor: browse a supplier catalog, add items to a cart, and generate a formal PO document.

Customer Evidence

"Select supplier, browse products with filters, add items to cart, generate PO." — Corey Shelson (Nov 2025 prototype)
"Take 6 material requests, generate 8 POs grouped by supplier." — Corey Shelson (auto-PO vision)
"You'll be able to cut purchase orders, receive shipments..." — DM Industries pricing call

2 · Scope ▼

In Scope

  • IN Extend PurchaseOrder model for supplier-specific fields (deliveryAddress, shippingMethod, expectedDeliveryDate)

  • OUT EDI/API transmission to suppliers
  • OUT PO approval workflow
  • OUT Mobile PO creation

4 · Data Model ▼

PurchaseOrder Extensions (type = SupplierPO)

The existing PurchaseOrder model (MAT-1) is extended with supplier-specific fields. The type discriminator distinguishes CustomerPO from SupplierPO.

FieldTypeNotes
deliveryAddressString? @db.TextJob site or warehouse address for delivery
shippingMethodString? @db.VarChar(100)e.g. “Will Call”, “Freight”, “Courier”
expectedDeliveryDateDateTime?Requested or promised delivery date

PurchaseOrderLineItem Extensions

FieldTypeNotes
materialIdString? @db.VarChar(21)FK → Material (from supply chain catalog)
manufacturerProductIdString? @db.VarChar(21)FK → ManufacturerProduct (specific product variant)
distributorPriceIdString? @db.VarChar(21)FK → DistributorPrice (locks in price at PO creation)

Line item total is computed: quantity × unitPrice. Pricing auto-populates from DistributorPrice.price minus any applicable discount (MAT-9).

Numbering

SPO-#### — auto-sequenced per instance. Follows the same counter pattern as existing entity numbering (Job, Invoice, etc.).

5 · API Surface ▼

Extended PurchaseOrder Type

extend type PurchaseOrder {
  deliveryAddress: String
  shippingMethod: String
  expectedDeliveryDate: DateTime
}

extend type PurchaseOrderLineItem {
  materialId: String
  Material: Material
  manufacturerProductId: String
  ManufacturerProduct: ManufacturerProduct
  distributorPriceId: String
  DistributorPrice: DistributorPrice
}

Input Types

input CreateSupplierPOInput {
  supplierId: String!
  jobId: String!
  deliveryAddress: String
  shippingMethod: String
  expectedDeliveryDate: DateTime
  notes: String
  lineItems: [SupplierPOLineItemInput!]!
}

input SupplierPOLineItemInput {
  materialId: String
  manufacturerProductId: String
  distributorPriceId: String
  description: String
  quantity: Float!
  unitPrice: Float
}

input UpdateSupplierPOInput {
  id: ID!
  deliveryAddress: String
  shippingMethod: String
  expectedDeliveryDate: DateTime
  notes: String
  lineItems: [SupplierPOLineItemInput!]
}

Queries & Mutations

type Query {
  supplierPurchaseOrders(options: GetListOptions): GetListResult!
  supplierPurchaseOrder(id: ID!): PurchaseOrder!

type Mutation {

  issueSupplierPO(id: ID!): PurchaseOrder!
  generatePurchaseOrderPdf(id: ID!): String!
  sendPurchaseOrderEmail(id: ID!, recipientEmail: String!): Boolean!
}

7 · Acceptance Criteria ▼

  • A Supplier PO can be created by selecting a distributor and a job
  • Line items auto-price from DistributorPrice when a product is selected from the catalog
  • Discounts from MAT-9 discount schedules are applied to line item pricing
  • PDF generates with company logo, PO number, supplier address, line items, subtotal/tax/total, terms, and signature line
  • Email sends the generated PDF to the supplier contact’s email address
  • PO status lifecycle: Draft → Issued → Partially Received → Fully Received → Closed
  • SPO numbering auto-sequences (SPO-0001, SPO-0002, …) per instance
  • Supplier PO tab appears on Company detail page showing all POs for that supplier
  • Permission checks enforce materialorder.create for PO creation and materialorder.edit for issuance
  • Validation: supplier is required, at least 1 line item required, quantity must be > 0
  • Committed costs from issued POs appear in the Job Financial Summary
  • PO list page supports filtering by type (Supplier PO), status, and supplier
  • Line item total computes correctly as quantity × unitPrice
  • PO detail page shows all line items with subtotals and grand total
