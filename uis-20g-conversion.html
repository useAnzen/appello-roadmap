1. Business Case ▼

The most painful step in the T&M workflow is the manual re-entry of work order data into change orders and invoices. Field workers capture labor, materials, and equipment on a work order, then office staff re-type the same information into a CO or invoice. This is error-prone, time-consuming, and the #1 source of billing disputes.

One-click conversion eliminates this friction entirely. Clearstory handles this by letting users select T&M tags and generating a fully-priced COR with a single button click. This is the critical bridge between field capture and office billing.

2. Scope ▼

In Scope

  • IN convertWorkOrderToChangeOrder mutation — creates ECO from WO line items
  • IN createInvoiceFromWorkOrder mutation — creates invoice from WO line items
  • IN sourceWorkOrderJobId FK on EstimateChangeOrderRecord for traceability
  • IN Confirmation modal with WO summary before conversion
  • IN Conversion status badges on WO detail page ("Converted to CO #CO-042")
  • IN "Convert to CO" and "Generate Invoice" buttons on WO detail header

Out of Scope

  • OUT Batch conversion (select multiple WOs and convert at once)
  • OUT Partial conversion (select a subset of line items to convert)

4. Data Model ▼

Schema Change: sourceWorkOrderJobId

Add a sourceWorkOrderJobId foreign key to EstimateChangeOrderRecord to link COs back to their source work order.

ModelFieldTypeDescription
EstimateChangeOrderRecord sourceWorkOrderJobId String? FK to Job (work order) that this CO was created from

Migration SQL

ALTER TABLE `EstimateChangeOrderRecord`
ADD COLUMN `sourceWorkOrderJobId` VARCHAR(191) NULL;

ALTER TABLE `EstimateChangeOrderRecord`
ADD CONSTRAINT `FK_ECO_SourceWorkOrder`
FOREIGN KEY (`sourceWorkOrderJobId`)
REFERENCES `Job`(`id`)
ON DELETE SET NULL;

7. Acceptance Criteria ▼

  • User can click "Convert to CO" on an approved work order and a confirmation modal appears with WO summary.
  • On confirmation, a new EstimateChangeOrderRecord is created with all labor, material, and equipment line items from the WO.
  • The created CO has sourceWorkOrderJobId set to the original WO job ID.
  • After conversion, the WO detail page shows a "Converted to CO #CO-XXX" badge and the "Convert to CO" button is disabled.
  • User can click "Generate Invoice" to create an invoice from WO line items.
  • Conversion buttons are only visible on WOs with "Approved" status.
  • Appropriate error messages display if conversion fails (e.g., WO has no line items).
