1 · Business Case & Customer Evidence ▼

Field workers consume materials daily but have no native way to log it from their phones. The current workaround is text messages, phone calls, or custom forms — none of which are linked to the material catalog or cost codes. This gap means consumption data arrives late, incomplete, and disconnected from the financial reporting chain. A mobile-first entry form closes the field-to-office loop, giving project managers real-time visibility into material burn and eliminating manual data re-entry.

2 · Scope ▼

In Scope

  • IN Mobile stepped form: select material, enter quantity, assign cost code, submit
  • IN Mobile entry list page per job (consumption history)
  • IN Integration into SimplifiedJobTasks (job site view) as a new task link
  • IN Uses existing MaterialConsumptionEntry model from MAT-2a
  • IN Uses ProgressionBar and SelectionSlideup patterns from existing mobile app

Out of Scope

  • OUT New Prisma models (uses MAT-2a models exclusively)
  • OUT Offline sync implementation (checklist item only — future phase)
  • OUT Desktop entry UI (covered by MAT-2a)
  • OUT Inventory deduction / stock level updates

4 · Data Model ▼

No new Prisma models are introduced. This slice consumes the MaterialConsumptionEntry model defined in MAT-2a. The mobile form maps to the create mutation fields as follows:

Mobile Form → MaterialConsumptionEntry Field Mapping

Mobile Form FieldModel FieldSource
Material Picker selection materialId SelectionSlideup → Material.id
Quantity stepper quantity User input (Decimal, must be > 0)
Unit (display only) unit Auto-populated from Material.unit
Cost Code dropdown costCodeId Job's assigned cost codes
Notes textarea notes User input (optional, VARCHAR 500)
(auto) jobId Current job context from route params
(auto) date Defaults to today, user can override
(auto) createdById Authenticated user.id

5 · API Surface ▼

No new API additions. The mobile app consumes the queries and mutations defined in MAT-2a.

Consumed Mutations

mutation CreateMaterialConsumptionEntry(
  $input: CreateMaterialConsumptionEntryInput!
) {
  createMaterialConsumptionEntry(input: $input) {
    id
    materialId
    quantity
    unit
    costCodeId
    notes
    date
    createdAt
  }
}

Consumed Queries

query MaterialConsumptionEntries($jobId: String!) {
  materialConsumptionEntries(jobId: $jobId) {
    id
    materialId
    quantity
    unit
    costCodeId
    notes
    date
    createdAt
    Material {
      id
      name
      unitCost
      unit
    }
    CostCode {
      id
      code
      name
    }
  }
}

query Materials($options: GetListOptions) {
  materials(options: $options) {
    data {
      ... on Material {
        id
        name
        unitCost
        unit
        category
      }
    }
  }
}

The materials query powers the material picker (Step 1). The materialConsumptionEntries query powers the history list page. The createMaterialConsumptionEntry mutation is called on form submit.

7 · Acceptance Criteria ▼

  • Stepped form completes: Step 1 (select material) → Step 2 (enter qty, cost code, notes) → Submit → confirmation
  • Material picker searches the company material catalog by name with results updating as the user types
  • Category tiles in the picker filter materials by their category field
  • Quantity input validates > 0; form cannot be submitted with zero or negative quantity
  • Cost code dropdown is optional and shows only cost codes assigned to the current job
  • Unit display (ft, sqft, each, etc.) is auto-populated from the selected material and is read-only
  • Estimated cost (qty × unit cost) updates in real time as quantity changes
  • Submitted entries appear in the consumption history list page for the job
  • Entries created on mobile sync to the desktop LEM report (visible in MAT-2a consumption table)
  • Permission gate: only users with materialorder.create permission can access the form
  • Offline behavior: entry is queued locally when no network connection, with a "Pending" badge shown in the list
  • Swipe-to-edit on the history list opens the form pre-populated with that entry's data
  • "Material Consumption" link appears in SimplifiedJobTasks and navigates to the list page
