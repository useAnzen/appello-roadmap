Material Billable Rates — UI

UI half of LEM-3. MaterialPricingCard on the Material detail page (cost / sell / margin + pricing-mode radios) and Default Markup % on the MaterialGroup add/edit form. Job Material Markup % is delivered by the DataModel SETTINGS form/view configured in LEM-3a — no JobDetailsPage code change here. Can run in parallel with WO-2/WO-3 once 3a is merged.

1. Scope

This spec covers LEM-3b only. Backend schema, service, and GraphQL resolvers are in LEM-3a and must be merged before this work starts. After 3a is in, this UI can run in parallel with WO-2/WO-3 since the fields and computed resolvers already exist on the GraphQL schema.

In Scope

  • IN Pricing card on the Material detail page showing cost, sell price, and margin
  • IN Pricing-mode radio buttons (group default / fixed sell price / custom markup %)
  • IN Default Markup % field on the MaterialGroup add/edit form

Out of Scope

  • OUT Schema migration, MaterialPricingService, GraphQL resolvers (covered by LEM-3a)
  • OUT Material Markup % field on Job — rendered automatically by the DataModel SETTINGS form / view-page section configured in LEM-3a. No JobDetailsPage code lives here.
  • OUT Any “materials assigned to a job” UI — that concept does not exist in the data model. The job-level markup applies to all materials priced against the job at line-item time; there is no curated material list per job.
  • OUT WO line item auto-populate from the cascade (handled by WO-2)
  • OUT Customer-level pricing tiers (future)

2. UI Specification

2a. Material Detail Page — Pricing Card

Add a “Pricing” card to the Material detail page showing cost, sell price, and margin. The user selects their pricing strategy via radio buttons.

  • Best Cost (read-only): resolved from DistributorPrice or MaterialAttributePrice
  • Sell Price (editable or computed): depends on the selected pricing mode
  • Margin (computed): dollar amount and percentage, shown inline

Pricing mode radio buttons:

  • Use group default — inherits MaterialGroup.defaultMarkupPercent
  • Fixed sell price — user enters sellPriceOverride
  • Custom markup % — user enters markupPercentOverride

2b. MaterialGroup Form — Default Markup

Add a Default Markup % field to the MaterialGroup add/edit form. This sets the Tier 1 default for all materials in the group.

2c. Job Settings — Material Markup Override (delivered by LEM-3a)

The Tier 3 job-level override is exposed automatically by the Job DataModel SETTINGS form and rendered on the Job Settings detail-page section configured in LEM-3a §6. Nothing to build here. The field appears as a percent input next to job-estimate-markup, job-overhead-markup, and hold-back-percent, and saves to Job.materialMarkupPercent.

2d. Downstream — WO Material Line Item Editor

When a user selects a Material from the catalog on a WO line item (handled by WO-2, listed here for context):

  • unitPrice (sell price) auto-populates from resolveMaterialSellPrice()
  • costPrice (cost) auto-populates from the cost basis
  • Both fields are editable — the user can override the auto-populated values
  • Margin is shown inline if the user has viewWageDetails permission

5. Acceptance Criteria

  • The Material detail page shows a Pricing card with cost, sell price, and margin ($ and %)
  • The Material detail page pricing mode radio buttons correctly switch between group default, fixed sell price, and custom markup %
  • Changing the pricing mode and saving persists the correct fields (sellPriceOverride or markupPercentOverride)
  • The MaterialGroup add/edit form includes a Default Markup % field
  • The Job Settings detail page renders the Material Markup field via the DataModel section-renderer (delivered by LEM-3a; no JobDetailsPage code change required here)
