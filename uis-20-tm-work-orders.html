1. Business Case

T&M Work Orders are the #1 missing feature across Appello's customer base and sales pipeline. Six customers have explicitly requested this capability, and one deal (Tear Systems) is directly blocked until it ships. The gap forces customers into form-based workarounds that lack structured line items, auto-pricing, and conversion workflows.

3. Architecture Overview

Key decision: A T&M Work Order is a type of Job (Tier 2 entity), not a new entity. This leverages everything Jobs already have: scheduling, timesheets, equipment, files, notes, permissions, financial summary, and the entire invoicing pipeline.

Data Flow

Field Worker (Mobile) → WorkOrderForm → Job (isWorkOrder=true)
    + WorkOrderLaborItem[] (auto-priced from TradeLevel + WageRegion)
    + WorkOrderMaterialItem[] (auto-priced from Material catalog)
    + WorkOrderEquipmentItem[] (auto-priced from Equipment catalog)
    + Signatures (geolocation-stamped PNG assets)
    + Photos (S3 assets via existing file system)

PM (Desktop) → Work Order Log → Review + Approve
    → Convert to Change Order (EstimateChangeOrderRecord)
    → Generate Invoice (JobBill)

PM (Desktop) → Change Order Log → Track aging + status across portfolio

Prerequisite

Emma's Job DataModel refactor (task/Q21030-11271/job-project-optional) must be merged first. It provides: projectId optional on Job, Job forms driven by DataModel, and job-level settings independent of project.
