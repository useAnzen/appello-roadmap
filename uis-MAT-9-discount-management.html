1 · Business Case & Customer Evidence ▼

Contractors negotiate discounts with suppliers — 15% off everything, 20% off insulation category, 25% off specific SKUs. Without structured discount tracking, PMs manually calculate effective prices on every order, introducing errors and masking true margins. A cascading discount model that resolves at the product level eliminates this friction and ensures every PO reflects the real negotiated price.

Customer Evidence

"Ingesting all of the Crossroads price books with all the discounts — that's going to be huge for us." — M-X Insulation
"Per-supplier discounts — we need to see the real price, not the list price." — Vanos Insulation (Nov 2025 prototype review)

Value Proposition

🎯
Three-Level Scoping
Global, Category, and Product discount scopes let contractors model the exact discount structure negotiated with each supplier.
⚡
Cascading Resolution
Most specific discount wins automatically — product-level overrides category, which overrides global. No manual override needed.
📅
Effective Date Ranges
Time-bound discounts with start/end dates. Expired discounts are retained for audit but excluded from price resolution.
📊
Margin Accuracy
Effective price flows into POs and P&L automatically, ensuring job costing reflects real negotiated rates.

Competitive Landscape

HAS ServiceTitan Vendor pricing tiers with multi-level discount support
PARTIAL Procore Contract pricing — limited to per-vendor flat discounts
HAS Sage Discount schedules with date ranges and tiered pricing
PARTIAL Appello (Current) discountPercent field on DistributorPrice only — no scoping or cascading

2 · Scope ▼

In Scope

  • IN DistributorDiscount Prisma model with scope enum (Global / Category / Product)
  • IN Discounts tab on Supplier Company detail page
  • IN Full CRUD for discounts (add, edit, deactivate)
  • IN resolveDiscount() algorithm — product > category > global (most specific wins)
  • IN resolveEffectivePrice() computed field on DistributorPrice
  • IN Effective dates with expired status display
  • IN Price Book Import can import discountPercent column

Out of Scope

  • OUT Volume-based discounts (buy X get Y%)
  • OUT Contract/agreement-level discount grouping
  • OUT Automatic discount application on invoices
  • OUT Mobile discount management

Discount Resolution Algorithm

async function resolveDiscount(
    distributorId: string,
    manufacturerProductId: string | null,
    categoryId: string | null
): Promise<Decimal | null> {
    const now = new Date();

    const discounts = await prisma.distributorDiscount.findMany({
        where: {
            distributorId,
            isDeleted: false,
            isActive: true,
            OR: [
                { effectiveFrom: null },
                { effectiveFrom: { lte: now } }
            ],
            AND: [
                {
                    OR: [
                        { effectiveTo: null },
                        { effectiveTo: { gte: now } }
                    ]
                }
            ]
        },
        orderBy: { createdAt: "desc" }
    });

    // 1. Product-specific discount (most specific)
    if (manufacturerProductId) {
        const productDiscount = discounts.find(
            (d) => d.scope === "Product"
                && d.manufacturerProductId === manufacturerProductId
        );
        if (productDiscount) return productDiscount.discountPercent;
    }

    // 2. Category-level discount
    if (categoryId) {
        const categoryDiscount = discounts.find(
            (d) => d.scope === "Category"
                && d.categoryId === categoryId
        );
        if (categoryDiscount) return categoryDiscount.discountPercent;
    }

    // 3. Global discount (least specific)
    const globalDiscount = discounts.find(
        (d) => d.scope === "Global"
    );
    if (globalDiscount) return globalDiscount.discountPercent;

    return null;
}

function resolveEffectivePrice(
    listPrice: Decimal,
    discountPercent: Decimal | null
): Decimal {
    if (!discountPercent) return listPrice;
    // effectivePrice = listPrice * (1 - discountPercent / 100)
    return listPrice.mul(
        new Decimal(1).minus(discountPercent.div(100))
    );
}

7 · Acceptance Criteria ▼

  • User can create a discount on a supplier company with name, scope, and discount percent
  • Scope selection works: Global shows no additional pickers, Category shows category picker, Product shows product picker
  • Category picker is displayed only when scope = Category
  • Product picker is displayed only when scope = Product
  • Discount resolution follows cascade: Product > Category > Global (most specific wins)
  • Expired discounts (effectiveTo < now) are shown with an “Expired” badge and are not applied to pricing
  • Effective price on DistributorPrice is computed correctly: listPrice × (1 − discountPercent / 100)
  • Price Book Import can ingest a discountPercent column and create Global-scope discounts
  • When no discount exists for a product, list price is used as effective price
  • Permission gate: pricing.edit is required to manage discounts (separate from material.edit)
  • Validation: discountPercent must be between 0 and 100 inclusive
  • Validation: name is required and cannot be empty
  • User can edit an existing discount (name, scope, percent, dates)
  • User can deactivate a discount (sets isActive = false)
  • Deactivated discounts are excluded from price resolution
